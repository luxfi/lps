# Lux Proposals (LPs)

Welcome to the Lux Proposals repository - the governance and standardization framework for the Lux Network blockchain ecosystem.

## What are LPs?

Lux Proposals (LPs) are technical specifications and standards that define protocols, interfaces, and processes for the Lux blockchain. Similar to Ethereum's EIP system, LPs provide a structured way for the community to propose and adopt improvements to the network.

## LP Categories

### Standards Track
Technical specifications requiring implementation:
- **Core**: Consensus, network rules, protocol changes
- **Networking**: P2P protocols, network layer
- **Interface**: APIs, RPC specifications
- **LRC**: Application standards (tokens, NFTs, DeFi)
- **Bridge**: Cross-chain protocols

### Other Types
- **Meta**: Process and governance proposals
- **Informational**: Guidelines and best practices

- **LRC-20**: Fungible Token Standard
- **LRC-721**: Non-Fungible Token Standard
- **LRC-1155**: Multi-Token Standard

## Getting Started

### Creating a New LP

1. Review existing proposals for similar ideas
2. Discuss your proposal on [forum.lux.network](https://forum.lux.network)
3. Use the [LP template](TEMPLATE) to structure your proposal
4. Submit a pull request with your draft LP

### LP Lifecycle

```
Draft → Review → Last Call (14 days) → Final
         ↓           ↓
    Withdrawn    Stagnant
```
